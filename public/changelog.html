Changelog

Retour au dashboard

Historique des evolutions du projet Webhook Discord.

v1.5.0 - Multi-pages + diagnostics logs

11 mars 2026
  • Separation landing/dashboard: nouvelle page /dashboard.html pour l'admin.
  • Callback OAuth redirige desormais vers le dashboard.
  • Collecte de logs persistants (API + worker) en fichiers ndjson dans logs/.
  • Endpoint diagnostics pour lire les derniers logs API depuis l'interface.
  • Ajout du request_id sur les erreurs API pour tracer rapidement les incidents.
  • Verification de presence bot durcie avec fallback de sonde guild.

v1.4.1 - Stabilite dashboard + dark mode polish

11 mars 2026
  • Compatibilite renforcee avec anciens schemas DB pour eviter les erreurs de chargement serveur.
  • Bascule theme avec icones lune/soleil au lieu du texte.
  • Amelioration des contrastes dark mode sur boutons, tableaux et blocs de contenu.

v1.4.0 - Routage par vehicule + UX dashboard

11 mars 2026
  • Regles de notification par vehicule (ex: 2439 + statut specifique, 2569 + autre statut).
  • Verification de presence bot plus robuste avec retries/delais pour limiter les faux negatifs.
  • Selection assistee des salons et roles via listes deroulantes quand le bot est present.
  • Lock temporaire du champ nom vehicule pendant la resolution auto pour eviter les conflits de saisie.
  • Ajout d'un dark mode sur le dashboard et enrichissement de la vitrine.

v1.3.0 - UI Redesign + UX

10 mars 2026
  • Landing page vitrine avant connexion avec bouton Discord OAuth.
  • Dashboard style Mee6 avec selection serveur visuelle.
  • Configuration vehicules en mode ID ou URL, avec recuperation automatique du nom.
  • Ajout de cette page changelog.

v1.2.0 - Flux vehicules flexibles

9 mars 2026
  • Support des entrees vehicules au format ID brut ou URL RSS complete.
  • Resolution automatique des informations vehicule via endpoint dedie.

v1.1.0 - Dashboard multi-serveur

9 mars 2026
  • Selection de serveur Discord admin et edition des regles de statut.
  • Sauvegarde des vehicules et regles par serveur.
